## Design Context

- Public experience stays minimal and functional, not marketing-heavy. Source: `01-CONTEXT.md` D-09, D-12.
- Auth uses separate login and signup pages. Source: `01-CONTEXT.md` D-01.
- Logged-in navigation stays top-nav only, with no sidebar in this phase. Source: `01-CONTEXT.md` D-11.
- Successful signup should feel like setup has started and route directly into onboarding. Source: `01-CONTEXT.md` D-04.
- Mobile responsiveness is required, but visual polish is MVP-grade rather than brand-heavy. Source: `REQUIREMENTS.md` UX-02 and project constraints.
- Landing-page display headline is the main focal point on public screens; the primary CTA is the secondary visual draw.
- On auth screens, the form card title is the focal point and the submit button is the secondary visual draw.

## Interaction Contract

- Public landing page presents one primary action (`Create account`) and one secondary action (`Log in`).
- Login and signup are separate routes with a single-column form-first layout on mobile and centered card layout on desktop.
- Protected-route failure redirects unauthenticated users to `/login` rather than rendering an inline access-denied screen.
- Successful signup routes directly to onboarding/profile setup.
- Authenticated users who have not completed setup are routed into onboarding, not a dashboard placeholder.
- Top navigation for authenticated pages includes only the minimum app identity, current destination, and sign-out access.
- Form validation should be inline and immediate on submit, with field-level errors placed below the affected field.
- Loading states use disabled primary actions with clear in-button progress text instead of spinners alone.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Create account |
| Empty state heading | No account session yet |
| Empty state body | Create an account or log in to start your workout setup. |
| Error state | We couldn’t sign you in. Check your email and password and try again. |
| Destructive confirmation | Sign out: Are you sure you want to sign out? |

Additional copy rules:
- Landing-page headline should describe the product in plain language, not marketing language.
- Signup success messaging should reinforce setup momentum, for example: `Account created. Let’s set up your plan.`
- Login page should prioritize return-to-flow language over feature explanation.
